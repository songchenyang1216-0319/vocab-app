# 四六级地铁背单词 App 产品需求文档

## 1. 产品目标

“四六级地铁背单词 App”是一个面向手机浏览器的轻量级背单词 Web App，帮助用户利用地铁、公交、排队等碎片时间学习四六级词汇。

第一版目标不是做一个复杂的学习平台，而是先做一个能稳定使用的单词卡片工具：

- 把项目中的 `CET4_CET6_5500_words_CN.md` 转换成 App 可读取的单词数据。
- 用户打开手机浏览器就能开始背单词。
- 用户可以按“认识 / 不认识 / 稍后复习”记录学习状态。
- App 自动保存学习进度，下次打开继续学习。
- 支持离线使用，适合在地铁信号不稳定的场景中使用。

## 2. 目标用户

主要目标用户：

- 正在准备大学英语四级或六级考试的学生。
- 想利用通勤时间背单词的人。
- 不想注册账号、不想安装大型 App、只想快速开始学习的用户。

用户特点：

- 每次学习时间较短，可能只有 3 到 10 分钟。
- 手机是主要使用设备。
- 使用环境可能比较嘈杂，网络可能不稳定。
- 更需要“快速进入学习状态”，而不是复杂的课程、社区或打卡系统。

## 3. 为什么适合在地铁上使用

地铁场景有几个明显特点：

- 时间碎片化：用户可能只坐几站，学习流程必须短。
- 单手操作：用户可能一只手扶栏杆，一只手拿手机。
- 网络不稳定：地铁里经常断网或弱网。
- 注意力容易被打断：到站、换乘、人群都会中断学习。
- 环境嘈杂：第一版不依赖音频，也不要求用户听发音。

因此 App 需要这样设计：

- 打开后直接进入学习，不需要登录。
- 页面按钮大，适合单手点击。
- 每次只展示一个单词，减少信息压力。
- 学习进度实时保存在本地。
- 支持离线缓存，断网也能继续背。
- 中断后再次打开，可以从上次位置继续。

## 4. 核心功能

### 4.1 单词学习

用户可以逐个查看单词卡片。卡片建议展示：

- 英文单词
- 中文释义
- 词性
- 所属范围：四级、六级、四/六、四级补充等
- 当前序号和总数

### 4.2 学习反馈

用户看完一个单词后，可以选择：

- 认识：表示这个词已经掌握得比较好。
- 不认识：表示这个词需要重点复习。
- 稍后复习：表示暂时不确定，后面再看。

第一版可以先用简单规则处理：

- 点“认识”后进入下一个词。
- 点“不认识”后把这个词加入复习列表。
- 点“稍后复习”后也加入复习列表，但优先级低于“不认识”。

### 4.3 今日学习

用户每天可以学习固定数量的单词，例如 20 个或 30 个。

第一版可以默认每天 20 个词，后续再增加设置项。

今日学习需要显示：

- 今天已学多少个
- 今日目标多少个
- 当前学习进度

### 4.4 复习

复习页展示用户标记为“不认识”或“稍后复习”的单词。

第一版复习逻辑可以保持简单：

- 不做复杂算法。
- 只把需要复习的词集中展示。
- 用户复习后可以再次标记为“认识”。

### 4.5 进度统计

首页或统计页需要显示基础学习数据：

- 总单词数
- 已学习单词数
- 已掌握单词数
- 待复习单词数
- 今日已学习数量

这些统计都来自本地保存的学习进度。

## 5. 页面设计

第一版建议包含以下页面。

### 5.1 首页

首页是用户打开 App 后看到的第一个页面。

主要内容：

- App 名称
- 今日学习进度
- 总体学习进度
- “开始学习”按钮
- “复习单词”按钮

首页不要做成复杂的宣传页，而应该像工具一样直接可用。

### 5.2 学习页

学习页是核心页面。

主要内容：

- 单词卡片
- 中文释义
- 标签，例如“四级”“六级”
- 当前进度，例如 `12 / 20`
- 操作按钮：认识、不认识、稍后复习

交互要求：

- 按钮要大，方便拇指点击。
- 一屏内尽量展示完整信息，避免频繁滚动。
- 用户点击反馈按钮后自动切换到下一个词。

### 5.3 复习页

复习页用于查看和复习用户还没掌握的单词。

主要内容：

- 待复习单词数量
- 当前复习单词卡片
- 操作按钮：已掌握、继续复习

### 5.4 统计页

统计页展示学习成果。

主要内容：

- 总学习进度
- 已掌握数量
- 待复习数量
- 连续学习天数，第一版可选

如果第一版想更简单，也可以先把统计信息放在首页，不单独做统计页。

### 5.5 设置页

设置页第一版可以很简单。

可选内容：

- 每日学习数量
- 清空本地学习进度
- 查看离线状态
- 查看词表版本

如果开发时间有限，设置页可以放到第二版。

## 6. 单词数据结构

原始词表文件格式示例：

```text
0001. **a** — art. 一（个）；每一（个） `[四级补充]`
0002. **abandon** — vt. 离弃，丢弃；遗弃，抛弃；放弃 n. 放任；纵情 `[四/六]`
```

App 不建议直接在运行时解析 Markdown。更适合在开发阶段把 Markdown 转换成 JSON 数据。

建议转换后的单词结构：

```ts
type Word = {
  id: number;
  word: string;
  phonetic?: string;
  partOfSpeech?: string;
  definitionCn: string;
  tags: string[];
  sourceLine: string;
};
```

字段说明：

- `id`：单词编号，例如 `1`。
- `word`：英文单词，例如 `abandon`。
- `phonetic`：音标，原始词表如果没有，可以先留空。
- `partOfSpeech`：词性，例如 `vt.`、`n.`、`art.`。第一版可以尽量解析，不完美也可以。
- `definitionCn`：中文释义。
- `tags`：标签，例如 `["四级"]`、`["六级"]`、`["四级", "六级"]`、`["四级补充"]`。
- `sourceLine`：原始 Markdown 行，方便以后检查解析结果。

转换后的 JSON 示例：

```json
{
  "id": 2,
  "word": "abandon",
  "partOfSpeech": "vt.",
  "definitionCn": "离弃，丢弃；遗弃，抛弃；放弃 n. 放任；纵情",
  "tags": ["四级", "六级"],
  "sourceLine": "0002. **abandon** — vt. 离弃，丢弃；遗弃，抛弃；放弃 n. 放任；纵情 `[四/六]`"
}
```

## 7. 学习进度如何保存

第一版不需要后端，也不需要登录。所有学习进度保存在浏览器本地。

可以使用 `localStorage`，优点是简单，适合新手开发。后续如果数据变复杂，再迁移到 `IndexedDB`。

建议保存的数据结构：

```ts
type WordStatus = "new" | "known" | "unknown" | "review";

type UserProgress = {
  version: number;
  lastStudyDate: string;
  currentIndex: number;
  dailyGoal: number;
  wordStatusMap: Record<number, WordStatus>;
  studiedWordIds: number[];
  reviewWordIds: number[];
};
```

字段说明：

- `version`：本地数据版本，方便以后升级数据结构。
- `lastStudyDate`：最近一次学习日期，例如 `2026-05-12`。
- `currentIndex`：当前学到词表中的第几个单词。
- `dailyGoal`：每日目标数量，例如 `20`。
- `wordStatusMap`：记录每个单词的状态。
- `studiedWordIds`：学过的单词编号。
- `reviewWordIds`：需要复习的单词编号。

建议 `localStorage` key：

```text
metro-vocab-progress
```

保存时机：

- 用户点击“认识 / 不认识 / 稍后复习”后立即保存。
- 用户修改每日目标后立即保存。
- 用户清空进度时删除对应本地数据。

## 8. 离线使用如何实现

技术路线使用 PWA，也就是 Progressive Web App。

PWA 可以让这个 Web App 像手机 App 一样被安装到桌面，并且可以缓存必要资源。

第一版离线能力需要缓存：

- HTML 页面
- JavaScript 和 CSS 文件
- 单词 JSON 数据
- App 图标
- PWA manifest 文件

建议使用 Vite 的 PWA 插件，例如 `vite-plugin-pwa`。

离线逻辑：

- 用户第一次联网打开 App 时，浏览器缓存 App 文件和词表数据。
- 之后即使地铁里没有网络，也可以打开已经缓存的 App。
- 学习进度保存在本地，不依赖服务器。

需要注意：

- 第一次使用必须有网络，因为浏览器还没有缓存 App。
- 如果后续更新词表或代码，需要处理缓存更新提示。
- 第一版可以先做简单更新策略：有新版本时提示用户刷新。

## 9. 第一版范围

第一版只做最核心的学习闭环。

### 第一版要做

- 使用 React、TypeScript、Vite 搭建项目。
- 把 Markdown 词表转换成 JSON。
- 手机优先的界面布局。
- 首页展示今日进度和总进度。
- 学习页展示单词卡片。
- 支持“认识 / 不认识 / 稍后复习”。
- 本地保存学习进度。
- 复习页展示需要复习的单词。
- 使用 PWA 支持安装和离线访问。

### 第一版不做

- 不做后端服务。
- 不做用户注册和登录。
- 不做云同步。
- 不做社交、排行榜、打卡分享。
- 不做复杂记忆算法。
- 不做付费功能。
- 不做多端同步。
- 不做发音音频。
- 不做例句、真题、词根词缀扩展。

这些功能不是不好，而是会明显增加开发难度。第一版应该先保证“能背、能保存、能离线”。

## 10. 后续可扩展功能

后续版本可以逐步增加以下功能。

### 10.1 更智能的复习算法

可以加入类似艾宾浩斯遗忘曲线的复习计划：

- 第 1 天复习
- 第 2 天复习
- 第 4 天复习
- 第 7 天复习
- 第 15 天复习

每个单词根据用户反馈计算下次复习时间。

### 10.2 单词筛选

支持按标签筛选：

- 只背四级词
- 只背六级词
- 只背四级补充
- 只复习不认识的词

### 10.3 搜索功能

用户可以搜索某个英文单词，查看释义和学习状态。

### 10.4 发音功能

可以使用浏览器自带的 Web Speech API 朗读单词。

第一版不做发音，是为了减少复杂度。后续可以在不引入后端的情况下逐步添加。

### 10.5 例句和笔记

未来可以给每个单词增加：

- 英文例句
- 中文例句翻译
- 用户自己的笔记
- 易混词提醒

### 10.6 数据导入和导出

为了避免用户换手机后丢失进度，后续可以支持：

- 导出学习进度 JSON 文件
- 导入学习进度 JSON 文件

这仍然可以不需要后端。

### 10.7 更好的统计

后续可以增加：

- 每日学习曲线
- 连续学习天数
- 掌握率
- 复习完成率
- 四级和六级分别的学习进度

### 10.8 IndexedDB 存储

如果后续数据变多，例如加入例句、音频缓存、用户笔记，可以从 `localStorage` 升级到 `IndexedDB`。

简单理解：

- `localStorage` 适合保存少量简单数据。
- `IndexedDB` 更像浏览器里的小型数据库，适合保存更复杂的数据。

## 推荐技术方案总结

第一版推荐技术路线：

- 前端框架：React
- 开发语言：TypeScript
- 构建工具：Vite
- 离线能力：PWA
- 本地存储：优先使用 localStorage
- 数据来源：从 `CET4_CET6_5500_words_CN.md` 转换出的 JSON 文件
- 后端：不需要
- 登录：不需要
- 部署方式：静态网站部署即可

这条路线适合新手，因为它把复杂度控制在前端范围内。只要静态文件能部署，用户就可以在手机浏览器中打开、安装、离线使用。
