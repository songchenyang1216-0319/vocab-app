# 四六级地铁背单词 App 使用说明

## 1. 怎么启动

先进入项目目录：

```bash
cd D:\vocab-app
```

安装依赖：

```bash
npm install
```

启动开发服务器：

```bash
npm run dev
```

启动后在浏览器打开终端里显示的本地开发地址即可。

开发模式适合写代码和本地调试，但 PWA 离线缓存通常要用打包后的预览或正式部署环境测试。

## 2. 怎么打包

执行：

```bash
npm run build
```

打包成功后会生成：

```text
dist/
```

`dist` 目录就是可以部署的静态网站文件。

当前构建会生成 PWA 文件：

```text
dist/manifest.webmanifest
dist/sw.js
dist/workbox-*.js
```

如果看到 bundle 超过 500KB 的提示，暂时不用紧张。当前词表 Markdown 被打进前端包里，所以体积会偏大；后续可以改成构建时生成 JSON 再优化加载。

## 3. 怎么部署

这个项目不需要后端，部署静态文件即可。

常见部署方式：

- Vercel
- Netlify
- GitHub Pages
- Cloudflare Pages
- 自己的 Nginx 静态服务器

部署时只需要把 `dist` 目录作为网站根目录。Vercel 是最省事的一种，建议优先用它。

部署前先运行：

```bash
npm run build
```

### 用 Vercel 部署

1. 把项目代码推送到 GitHub 仓库。
2. 打开 Vercel，点击 `New Project`。
3. 导入这个 GitHub 仓库。
4. 保持默认构建设置即可：
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. 点击部署，等待 Vercel 生成线上地址。

Vercel 会自动使用 HTTPS，适合手机直接访问，也支持 PWA 的 service worker 和添加到主屏幕。

### 手机上怎么访问线上地址

部署成功后，Vercel 会给你一个线上地址，通常长这样：

```text
https://你的项目名.vercel.app
```

在手机浏览器里直接打开这个地址即可。建议第一次先联网打开一次，让 PWA 缓存好核心资源。

### 本地预览生产包

如果你想在本地先看一下打包后的效果，可以运行：

```bash
npm run preview
```

然后打开终端里显示的本地预览地址。

PWA 在正式使用时建议部署到 HTTPS 地址。手机浏览器通常要求 HTTPS 才能完整支持 service worker 和添加到主屏幕。

## 4. 手机怎么使用

手机打开部署后的 HTTPS 地址。

首页可以看到：

- 总词数
- 今日已背
- 累计认识
- 错词数量
- 生词本数量

主要页面：

- `开始背单词`：每次显示一个英文单词，点击“显示释义”后查看中文。
- `复习错词`：复习已经到期的单词，每次显示一个英文单词，点“显示释义”后再选择“记住了 / 还模糊 / 没记住”。
- `生词本`：集中查看你手动收进去的单词，支持移出生词本。
- `搜索单词`：可以输入英文或中文释义搜索。
- `设置`：修改每日目标、背诵范围、背诵顺序，导入或导出学习记录。

今日任务规则：

- 每天第一次打开 App 时，会自动生成当天任务。
- 今日任务由两部分组成：`今日复习` 和 `今日新词`。
- 如果有到期复习词，`开始背单词` 会先进入复习。
- 今日复习完成后，页面会自动切换到今日新词。
- 当天的新词数量和复习上限可以在设置页调整。
- 今日任务全部完成后，会显示“今日任务已完成”。
- 今日任务完成后，仍然可以点击 `继续背新词` 再追加一批新词。
- 如果你怀疑任务生成不对，可以点击 `重新生成今日任务`。
- 设置页里的背诵范围会直接影响今日任务生成；如果当前范围没有可用单词，背单词页会提示你先去调整范围。

背单词页规则：

- 点击首页“开始背单词”会从上次中断的位置继续。
- 背单词入口会优先完成今天到期的复习词，再继续今日新词。
- 点击“认识”后，会先显示当前单词释义，并出现 `记错了` 和 `下一个` 两个按钮。
- 如果发现自己刚才记错了，可以点击 `记错了`，系统会按 `vague` 记录，并进入下一个单词。
- 如果确认自己确实认识，可以点击 `下一个`，系统会按 `known` 记录，并进入下一个单词。
- 点击“模糊”后，会先显示当前单词释义，再让你做二次判断：`不认识` 或 `有印象`。
- 在“模糊”的二次判断里，点击“有印象”会按 `vague` 记录，并直接进入下一个单词。
- 在“模糊”的二次判断里，点击“不认识”会按 `unknown` 记录，并直接进入下一个单词。
- 点击“不认识”后，会先按 `unknown` 记录，再显示释义，并提供 `加入生词本` 和 `下一个` 两个操作。
- 点击“不认识”后的 `加入生词本`，会把当前单词加入生词本；如果已经在生词本里，不会重复添加。
- 在等待自动跳转或等待二次选择时，页面不会继续显示“认识 / 模糊 / 不认识”按钮，避免重复计数。
- “认识 / 模糊 / 不认识”按钮在手机上按下时会有明显反馈。

生词本页面规则：

- 生词本按 `wordId` 保存，不重复保存整条释义数据。
- 页面会显示英文单词、中文释义、标签和加入时间。
- 点击“移出生词本”后，只会从生词本里移除，不会删除学习记录。

复习页规则：

- 只显示 `nextReviewAt` 已经到期的单词。
- 错误次数越高越优先复习。
- `unknown` 状态优先于 `vague` 状态。
- 如果没有到期单词，会显示“今天没有需要复习的单词”。
- 点击“记住了”后，下次复习时间会设置为 3 天后。
- 点击“还模糊”后，下次复习时间会设置为明天，并增加错误次数。
- 点击“没记住”后，下次复习时间会设置为 30 分钟后，并增加错误次数。

添加到主屏幕：

Android Chrome：

1. 用 Chrome 打开 App 地址。
2. 点击右上角菜单。
3. 选择“添加到主屏幕”或“安装应用”。
4. 桌面会出现“背单词”。

iPhone Safari：

1. 用 Safari 打开 App 地址。
2. 点击底部分享按钮。
3. 选择“添加到主屏幕”。
4. 桌面会出现“背单词”。

如果使用的是 Vercel 线上地址，先正常打开网页一次，再执行添加到主屏幕，这样图标和离线缓存更稳定。

离线使用：

1. 第一次必须联网打开 App。
2. 等页面加载完成后，PWA 会缓存核心资源。
3. 下次断网时，可以再次打开已经缓存过的 App。
4. 学习记录保存在本机浏览器的 `localStorage`，不需要登录。

### 如何更新线上版本

1. 修改代码后提交并推送到 GitHub。
2. Vercel 会自动重新构建并发布新版本。
3. 用户下次打开 App 时，如果 service worker 发现新版本，会提示刷新。
4. 如果你想尽快看到新版本，可以在手机浏览器里手动刷新一次。

## 5. 如何更新词表

当前 App 使用的词表文件是：

```text
src/data/CET4_CET6_5500_words_CN.md
```

词条格式必须保持类似：

```text
0001. **a** — art. 一（个）；每一（个） `[四级补充]`
0002. **abandon** — vt. 离弃，丢弃；遗弃，抛弃；放弃 n. 放任；纵情 `[四/六]`
```

支持的标签只有：

```text
四/六
六级
四级补充
```

更新步骤：

1. 替换或编辑 `src/data/CET4_CET6_5500_words_CN.md`。
2. 保持每个单词一行，行首使用 4 位编号。
3. 运行构建检查：

```bash
npm run build
```

4. 如果构建通过，再重新部署 `dist` 目录。

如果词表格式错误，解析函数会在浏览器控制台给出清晰提示。可以先打开：

```text
/vocab-preview
```

查看前 20 个解析结果，并确认总数量是否接近 5500。

## 6. 本地数据说明

学习记录保存在：

```text
metro-vocab-progress-v1
```

设置项保存在：

```text
metro-vocab-settings-v1
```

今日任务保存在：

```text
daily-study-task-v1
```

生词本保存在：

```text
vocab-book-items
```

这些数据都在当前浏览器本地。换浏览器、清除浏览器数据或换手机后，本地记录不会自动同步。

建议定期在设置页点击：

```text
导出学习记录 JSON
```

需要恢复时，在设置页点击：

```text
导入学习记录 JSON
```

如果只想清空生词本，可以去设置页的数据管理区域点击：

```text
清空生词本
```

这个操作不会删除学习记录，只会移除生词本里的单词。
